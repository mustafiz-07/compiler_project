# CyberLang Compiler

A cyberpunk-themed compiler for CyberLang, a C-variant language.  
Produces generated **C source (`output.c`)** from `.cyber` source files.

---

## Files

| File | Description |
|---|---|
| `cyberlang.l` | Flex lexer ‚Äì tokenises all CyberLang keywords, operators, and literals |
| `cyberlang.y` | Bison parser ‚Äì full grammar + AST-style semantic actions + C emission + symbol table |
| `Makefile` | Build automation |
| `test.cyber` | Sample program exercising every language feature |

---

## Prerequisites

```
sudo apt install flex bison gcc   # Debian/Ubuntu
brew install flex bison gcc       # macOS
```

---

## Build

```bash
make
```

This runs:
1. `bison -d cyberlang.y` ‚Üí `cyberlang.tab.c` + `cyberlang.tab.h`
2. `flex cyberlang.l`     ‚Üí `lex.yy.c`
3. `gcc`                  ‚Üí `./cyberlang`

---

## Run

```bash
# Compile the sample program and view generated C
make run

# Or pipe any .cyber file manually
./cyberlang < test.cyber

# `output.c` is generated by the compiler
./cyberlang < test.cyber
```

### Data Types
| CyberLang | C equivalent |
|---|---|
| `byte` | `int` |
| `deci` | `float` |

| `megabyte` | `long int` |
| `symbol` | `char` |

| CyberLang | Meaning |
|---|---|
| `cyber` | function declaration |
| `bootup` | main / entry point |
| `green` | if |
| `yellow` | else if |
| `red` | else |

| `round` | for |
| `execute` | do (do-while) |

| `node` | case |
| `hack` | break |
| `skip` | continue |

| Function | Description |
|---|---|

## Compiler Features

### Symbol Table
- Scope-aware (enter/exit scope on every `{ }` block)
- Redeclaration warning in same scope
- Undeclared-use error

### C Generation
- Temporary variables: `t0`, `t1`, ‚Ä¶
- Labels: `L0`, `L1`, ‚Ä¶
- All control flow constructs produce correct jump-based C flow
- For-loop update expression is **deferred** so it appears after the loop body

### Constant Folding Optimisation
Integer arithmetic expressions where **both operands are compile-time constants** are folded at compile time ‚Äî no temporary variable is emitted:

```
folded << 3 + 4 + 2 + 2;
// Generated C still folds to: folded = 11
```
